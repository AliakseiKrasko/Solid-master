# Solid-master
Мы рефакторировали проект, чтобы он соответствовал принципам SOLID. Вот основные изменения:

SRP: Разделили обязанности между классами FamilyTree и FileManager.
OCP: Добавили новый класс MemberSorter для расширения функциональности без изменения существующего кода.
LSP: Убедились, что подклассы могут заменять базовые классы.
ISP: Создали отдельные интерфейсы для различных функциональностей.
DIP: Сделали классы независимыми от конкретных реализаций и зависимыми от абстракций.
